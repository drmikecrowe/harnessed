<objective>
Add the credential-free scan gate and the raw-npm/npx recipe lint, wire both into `harnessed build` so it
vetoes bad input and bad dependencies before anything is committed or baked (BLD-02 + BLD-03).

Purpose: This completes the "trustworthy build" goal. `harnessed build` today emits a profile and bakes the
hatago image with zero supply-chain checks. BLD-03 makes the assembler reject recipes that reach for `npm`/`npx`
(with the pnpm equivalent named); BLD-02 makes `harnessed build` fail when a vendored/source/Python dependency
or a baked image carries a HIGH+ vulnerability. The crux (RESEARCH Pitfall 3): osv-scanner `scan` exits 1 on
*any* finding with no severity flag — so the HIGH threshold is Python logic over `--format json`, not a flag.
The gate's severity-awareness is proven not only by a HIGH-abort fixture but by a LOW/MEDIUM fixture that must
build GREEN — the only test that distinguishes a real CVSS gate from a naive exit-1-as-high gate.

Output: `tools/harnessed/scan.py` (new gate module, scoped source scan + image scan) + osv-scanner/pip-audit in
the tools image (offline DB pre-seeded) + `scan`/`scan-image` subcommands + `validate_no_raw_npm()` in
schema/assemble + build_stack wiring (scoped source scan + host image scan) + a `--root` override on the
launcher + integration fixtures (npm/vuln/low) in the resolver layout.
</objective>
