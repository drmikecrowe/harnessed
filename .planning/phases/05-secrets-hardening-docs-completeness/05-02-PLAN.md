## Phase Goal

**As a** stack operator, **I want to** opt into 1Password-backed secrets via a `.env.schema` and set scanner tokens with a single command, **so that** my stacks gain hardened secrets handling without regressing the zero-config default, and secrets/tokens never touch an image layer or committed file.

## Phase User Story (MVP vertical slice — SEC-01 + SEC-03)

This plan delivers two user-facing capabilities: (1) opt-in secret resolution (SEC-01) — with a `.env.schema`, secrets reach the pod; without it, nothing changes; (2) one-shot scanner-token auth (SEC-03) — `harnessed auth snyk|socket` persists a token to host config. The secrets setup doc ships here per the design §17 cadence rule (a reader of SEC-01 has the doc the moment the feature lands).

<objective>
Create `lib/harnessed-secrets.sh` with `resolve_secret_env` (the opt-in detect-and-resolve primitive — throwaway tools container runs `varlock load`, host captures resolved env into a mode-0600 temp file) and `auth_scanner` (the `harnessed auth snyk|socket` handler — `--rm` tools container drives the vendor CLI's own auth with `~/.config` rw-mounted). Wire `resolve_secret_env` into `harnessed_isolated` before pod member creation, pass the resolved env to both members via `--env-file`, and unlink the temp file after launch. Add the `auth` subcommand dispatch to the launcher. Ship `docs/guides/secrets.md` with SEC-01 (cadence).

Purpose: SEC-01 is the opt-in secrets layer — the `.env.schema` is the switch, varlock + the already-mounted 1Password agent socket (`lib/harnessed-mounts.sh:23-27`) are the transport, and `--env-file` is the injection (env only, never baked). SEC-03 is the token-setup companion — `harnessed auth` drives the vendor CLI's own persistence (`snyk config set api=`/`socket login`) so the token lives in host config, never an image layer. The inertness guarantee (no schema ⇒ no varlock) is non-negotiable.

Output: New `lib/harnessed-secrets.sh`, wired `lib/harnessed-isolated.sh` + `harnessed` launcher, new `docs/guides/secrets.md`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host filesystem → throwaway tools container | The `.env.schema` + the 1Password agent socket cross here (ro mounts); resolved env crosses back via the temp env-file |
