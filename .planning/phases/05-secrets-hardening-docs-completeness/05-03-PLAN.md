## Phase Goal

**As a** stack operator, **I want** a nightly timer to re-scan my installed images for newly-disclosed CVEs, **so that** a vulnerability published after my build is caught automatically rather than going unnoticed until the next manual build.

## Phase User Story (MVP vertical slice — SEC-04)

This plan delivers the nightly re-scan timer end-to-end: the online image-scan function (fresh DB, the whole point), the `harnessed rescan` subcommand that iterates installed images, the static systemd user-timer units, and the launcher dispatch.

<objective>
Add `run_image_scan_online` to `scan.py` (the online variant of the existing offline `run_image_scan` — drops `--offline --offline-vulnerabilities` so osv-scanner sees newly-disclosed advisories). Expose it via a `scan-image-online` CLI subcommand. Create `lib/harnessed-rescan.sh::harnessed_rescan_images` (iterates harnessed-labelled images, `podman save`s each, scans online). Add the `rescan)` dispatch to the launcher. Ship static systemd user-timer units (`systemd/harnessed-rescan.{timer,service}`) the operator copies to `~/.config/systemd/user/`.

Purpose: SEC-04 closes the time-of-check/time-of-use gap — the build-time gate (Phase 3) uses the offline DB (deterministic), so it only knows about CVEs that existed at build time. The nightly re-scan uses the ONLINE DB to catch post-build disclosures. The critical detail (Pitfall 6): the nightly MUST NOT use the offline flags, or it sees nothing new. The `loginctl enable-linger` prerequisite (Pitfall 5 — currently OFF on the host) is a documented setup step, not a blocker.

Output: Extended `scan.py` + `cli.py`, new `lib/harnessed-rescan.sh`, launcher dispatch, new `systemd/` units.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| systemd user timer → `harnessed rescan` | The timer fires `ExecStart=%h/.local/bin/harnessed rescan` as the user (rootless, no root) |
| host `podman save` → tools container `scan-image-online` | The saved image tar crosses here (ro mount); the tools image scans it ONLINE against osv.dev |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | Tampering/Time-of-check | Post-build CVE goes unnoticed (the build-time offline DB doesn't know about it) | mitigate | Nightly ONLINE osv-scanner re-scan (`scan-image-online`, no `--offline` flags) against every installed harnessed-labelled image; `Persistent=true` catches missed runs after boot. |
| T-05-12 | Operational (→ Tampering window) | Timer silently never fires (user logged out, `loginctl enable-linger` OFF) | mitigate | Document `loginctl enable-linger $USER` as a HARD prerequisite in the timer units' comments + the troubleshooting doc (05-04); `Persistent=true` in the timer; troubleshooting lists the diagnostic (`systemctl --user list-timers` + `journalctl --user -u harnessed-rescan.service`). |
| T-05-13 | Denial of Service | Online scan hangs on network failure to osv.dev | mitigate | `_TIMEOUT = 300` (scan.py:34, existing) bounds `run_image_scan_online`; a network failure surfaces as exit 128 / non-clean (investigate), not a silent pass; each image is scanned independently (one failure doesn't abort the set). |
| T-05-14 | Elevation of Privilege | Timer runs as root (system unit) | mitigate | The units are USER units (`~/.config/systemd/user/`), NOT system units — rootless, scoped to the user's UID, consistent with rootless podman. |
| T-05-15 | Information Disclosure | Re-scan uploads image contents to a third party | mitigate | osv-scanner scans the LOCAL `podman save` tar against osv.dev's advisory API (manifest hashes, not full image upload); no Socket.dev (server-side) in the nightly — only the credential-free osv-scanner. |

    ```bash
    local img tar rc=0 img_rc
    for img in $("$CONTAINER_RUNTIME" images --filter reference='harnessed-*' \
                 --format '{{.Repository}}:{{.Tag}}'); do
        [ -n "$img" ] || continue
        print_info "Re-scanning $img (online) ..."
        tar="$(mktemp --suffix=.tar)"
        "$CONTAINER_RUNTIME" save "$img" -o "$tar"
        img_rc=0
        "$CONTAINER_RUNTIME" run --rm -v "$tar":"$tar":ro \
            "$HARNESSED_TOOLS_IMAGE" scan-image-online "$tar" || img_rc=$?
        rm -f "$tar"
        if [ "$img_rc" -ne 0 ]; then
            print_error "Re-scan of $img found a HIGH+ finding (newly-disclosed CVE) — investigate"
            rc=1   # track overall failure; do NOT abort the loop (scan remaining images)
        fi
    done
    return "$rc"
    ```
