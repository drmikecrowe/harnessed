# Phase 02 / Plan 01: Recipe+Stack Schema & Emit-Only Assembler Summary

**The reproducible build pipeline: recipe/stack YAML schema, the emit-only `harnessed-tools` Python assembler (fail-fast collisions), base/Dockerfile.hatago, and a `harnessed build <stack>` subcommand — producing a committed `profiles/tracer-time/` tree wired to a single hatago endpoint.**

## Performance

- **Duration:** ~35 min
- **Tasks:** 3 auto-tasks (+ 1 blocking human-verify checkpoint, deferred to operator)
- **Files modified:** 19 (17 created, 2 modified)

## Accomplishments
- Recipe + stack schema (design §11/§12) with the `tracer-time` instance: `claude` harness + `time` recipe (one `uvx mcp-server-time` stdio server + one standalone `time-helper` skill).
- `harnessed-tools` emit-only assembler: parses/validates YAML (tolerant of unknown fields, D-14), fans skills/commands into the harness profile, merges MCP servers into `hatago.config.json`, derives the single-endpoint `.mcp.json`, and records the baked-server manifest — never invoking podman/docker.
- Fail-fast collision: a duplicate skill/command name aborts the build (non-zero) naming BOTH colliding source paths (RCP-04, D-13).
- `base/Dockerfile.hatago` bakes the hatago hub (`pnpm add -g`, pinned) + `uvx mcp-server-time` (pinned) so hatago runs it as an offline stdio→HTTP child (MCP-03).
- `harnessed build <stack>` dispatches emit-only assemble → host `podman build` of the hatago image, while bare `harnessed build`/`--build` still rebuild the base/claude/hatago images.
- Committed `profiles/tracer-time/` tree (the artifact plan 02-02 mounts) generated by running the assembler locally.
