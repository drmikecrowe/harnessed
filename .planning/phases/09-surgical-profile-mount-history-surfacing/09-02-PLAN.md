<objective>
Refactor the Python assembler (emit.py + assemble.py) so that build output targets
profile_dir root instead of the .claude/ subdirectory, and remove the fan-out step
that wrote skills/commands/agents/hooks/rules into profiles/<stack>/.claude/.

Purpose: The assembler's old model wrote a whole .claude/ tree to profiles/<stack>/
so the launcher could mount it. Phase 9 eliminates that model: skills are baked into
the harness image at build time (per D-11), and only .mcp.json + settings.json land at
profile root. This is the assembler side of MNT2-01.

Output: Modified emit.py (no ensure_profile_tree, no PROFILE_SUBDIRS; write_mcp_json /
write_settings_json emit to profile_dir); modified assemble.py (no syncer fan-out, no
harness_dir, emit targets redirected to profile_dir).
</objective>
