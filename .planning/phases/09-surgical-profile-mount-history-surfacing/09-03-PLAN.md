<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Launcher → container | All -v and -w flags flow from launcher code; wrong paths expose host filesystem segments to the container |
| Profile dir → is-built guard | Guard checks .mcp.json existence at profile root; old .claude/-based guard would pass stale profiles |
| Project path → workdir | $project_path is set from $PWD (host) — malicious working directories with special chars could affect bind mount specs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-B01 | Information Disclosure | Residual .claude/ mounts from old copy block | mitigate | Old copy-and-mount block removed in Task 1; replaced by harnessed_manifest_mounts which mounts only manifest-listed paths; no whole-dir .claude/ bind possible after this change |
| T-09-B02 | Tampering | mcp_cfg path update (Pitfall 1) | mitigate | mcp_cfg changed to $CONTAINER_HOME/.mcp.json; claude --mcp-config points to the ro-mounted file from profiles/<stack>/.mcp.json; no write access to this file from the container |
| T-09-B03 | Tampering | workdir set to $project_path (MNT2-02) | accept | $project_path derives from $PWD which is set by the caller (the user); no different exposure than the existing project bind mount; the same path is already bind-mounted as the project volume |
| T-09-B04 | Elevation | stale profile passes new is-built guard | mitigate | D-10 guard checks [ -f "$profile_dir/.mcp.json" ]; stale profiles (with .claude/ but no .mcp.json at root) fail the guard and force rebuild per D-09 |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
