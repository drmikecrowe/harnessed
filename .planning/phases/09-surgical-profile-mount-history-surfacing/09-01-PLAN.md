<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Manifest YAML → mount decisions | yq parses committed repo files; malformed YAML or injected paths could affect mounts |
| Host $HOME → container via history rw-mounts | rw bind allows container writes to land on host; wrong paths expose credentials |
| profile_dir → container ro-mounts | profile files are repo-committed config; wrong target paths could overwrite image-baked config |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Information Disclosure | history dir rw-mounts (MNT2-03/04/05) | mitigate | Manifests list specific named subdirs only; parent dirs (.claude/, .omp/, .gemini/) never appear in history_dirs; auth files (agent.db, antigravity-oauth-token, .credentials.json) explicitly excluded by manifest content policy enforced during this task |
| T-09-02 | Tampering | omp slug derivation | mitigate | Slug computed from HOST $relpath using the same algorithm omp uses; pre-create host dir with mkdir -p before bind so podman does not create it root-owned (DooD) |
| T-09-03 | Information Disclosure | antigravity history paths | mitigate | Manifests mount only .gemini/antigravity-cli/{conversations,brain,implicit} — never .gemini/ proper (Pitfall 6 enforced at manifest authoring time) |
| T-09-04 | Tampering | malformed manifest YAML | accept | yq returns empty on parse error (2>/dev/null); no entries → no mounts; safe default (no mount is always safer than wrong mount) |
| T-09-SC | Tampering | npm/pip/cargo installs | accept | No new packages installed in this plan |
</threat_model>
