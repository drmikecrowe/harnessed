## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| build-time image pull | ubuntu:24.04 + apt packages + mise downloads from external registries during `podman build` |
| mise runtime installs | bun/rust/go/node downloaded by mise from their upstream registries at build time |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-01 | Tampering | mise use -g node/bun/rust/go downloads | mitigate | Existing osv-scanner image scan gate (BLD-02) runs over the built image after assembly; catches known CVEs in baked layers |
| T-07-02 | Spoofing | curl https://mise.run in Dockerfile | accept | mise is an established tool (jdx.dev); the `https://mise.run` installer is the canonical install path used in v1; switching to a pinned binary would be Phase 8+ pinning work |
| T-07-SC | Tampering | npm/pip/cargo installs | accept | No new npm installs; existing @openai/codex and @google/gemini-cli remain and were already in the image; existing supply-chain gate covers the resulting image |
</threat_model>
