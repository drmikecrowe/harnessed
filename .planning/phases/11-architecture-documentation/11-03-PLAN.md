Append a new "Worked example 3: a Dockerfile recipe (installs a framework CLI)" section after the existing "Worked example 2" and before the "Transports" section. The new section must show:

1. **The recipe.yaml** for a framework recipe (model it on recipes/gstack/recipe.yaml):
   ```yaml
   name: gstack
   description: Installs the gstack tooling via its framework installer.
   harnesses: [claude]
   expect: [gstack-skill]
   ```
   Explain each field: `harnesses: [claude]` means this recipe is claude-only; composing it onto an omp stack is a validation error. `expect: [gstack-skill]` tells the capability test to confirm `gstack-skill` is present in the running instance.

2. **The Dockerfile** (model it on recipes/gstack/Dockerfile):
   - No `FROM` line (the assembler supplies `FROM harnessed-${HARNESS}:latest`)
   - `ARG HARNESS=claude` at the top (lets the body reference `${HARNESS}` for harness-aware installs; stripped by assembler during concatenation)
   - A pinned install step using `pnpm dlx @gstack/install@1.2.3 --host ${HARNESS}` — this is the "run the framework's own installer" principle: let the framework install itself, pinned to an exact version
   - A note that floating refs (`--branch main`, `@latest`) are rejected by the assembler's pin validation

3. **"Run the framework's own installer" principle**: Add a short paragraph explaining that recipe Dockerfiles do not manually copy files or vendor deps — they invoke the framework's published installer (e.g., `pnpm dlx @framework/install@<version> --host ${HARNESS}`). The `--host ${HARNESS}` flag lets the installer configure itself for the target harness. This keeps recipes thin and delegates version management to the framework.

4. **Pin discipline**: Note that `ARG` declarations in the recipe Dockerfile are the standard way to express version pins. The assembler validates that every downloadable resource has a pinned version — no floating `@latest` or `--branch main`.

5. **Lifecycle**: End the section with the full build-and-test lifecycle:
   ```bash
   harnessed build gstack-time   # assembles gstack + time recipes, scans derived image
   harnessed gstack-time         # launch
   harnessed test gstack-time    # capability report: ✓ gstack-skill (expect), ✓ time (mcp)
   ```

Do NOT change the time or ping worked examples.
