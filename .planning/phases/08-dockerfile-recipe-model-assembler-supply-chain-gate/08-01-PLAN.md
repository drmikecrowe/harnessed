<objective>
Extend the Python assembler module with three capabilities: (1) recipe schema extensions for
`harnesses:` and `expect:` fields plus pre-emission validation functions; (2) a
`write_derived_dockerfile()` emitter that produces the `profiles/<stack>/Dockerfile.harnessed-<stack>`
artifact; (3) a `run_snyk_container_scan()` function and `scan-snyk-container` CLI subcommand.

Purpose: These are the pure-Python building blocks that the rest of Phase 8 wires together. All
three ship in one plan because they are tightly coupled (schema → assemble → emit → scan → cli)
and none touches bash or container builds.

Output: Five modified Python files; `harnessed build` can now validate harness compat + floating
refs and emit a derived Dockerfile during the assemble step.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| recipe author → assembler | Recipe Dockerfile content is untrusted input; assembler validates before emitting |
| assembler → host podman build | Emitted Dockerfile is machine-generated; host runs `podman build` on it |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering | recipe Dockerfile floating ref (`--branch main/master/HEAD`, `:latest`, `@latest`) | mitigate | `validate_pin()` + `_FLOATING_REF_RE` regex gate in `assemble()` pre-emission (ASM-02); `PinValidationError` raised before any file is written |
| T-08-02 | Spoofing | incompatible harness composition (claude-only recipe on omp stack) silently produces wrong image | mitigate | `validate_harness_compat()` gate in `assemble()` pre-emission (ASM-01); `HarnessCompatError` raised before any file is written |
| T-08-03 | Tampering | raw npm/npx in recipe Dockerfile bypasses pnpm supply-chain policy | mitigate | existing `validate_no_raw_npm()` gate (BLD-03) in same loop; continues to enforce pnpm-everywhere |
| T-08-SC | Tampering | npm/pip/cargo installs in recipe Dockerfile bodies | mitigate | slopcheck: pin validation (T-08-01) blocks floating refs; pnpm-everywhere lint (T-08-03) blocks raw npm; no new packages installed in this plan |
</threat_model>
