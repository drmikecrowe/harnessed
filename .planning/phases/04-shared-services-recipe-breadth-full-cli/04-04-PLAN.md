### Task 1 — No-args shows help (gap 6B)

**Source:** `04-UAT.md` gap 6B · red test `test_no_args_help`.

**Change** (`harnessed`, after the arg-default block at line ~80, before the `while [[ $# -gt 0 ]]` parse loop):

```bash
# Bare invocation (no args) shows help instead of silently launching the transparent
# stack (UAT gap 6B). A single bareword (e.g. a project path) still launches transparent —
# the guard is on $# == 0, not on the STACK default.
[ $# -eq 0 ] && { usage; exit 0; }
```

**Why `$# == 0`, not STACK default:** `harnessed <path>` (one bareword that is a project path)
must still launch transparent; only a truly empty invocation routes to help.

**Done when:** `test_no_args_help` passes (bare `harnessed` exits 0 and output contains "Usage").

### Task 2 — Legible state-dir slug (gap 6)

**Source:** `04-UAT.md` gap 6 · red test `test_legible_slug`.

**Change** (`lib/harnessed-isolated.sh`, the `run_claude` construction at line ~103):

```bash
# state dir keyed by a LEGIBLE flattened project path + stack (UAT gap 6) — NOT the
# opaque hash instance name ($instance still keys the pod/container name above).
local state_project="${relpath//'/'/-}"   # project_relpath() is home-relative
local run_claude="${XDG_STATE_HOME:-$HOME/.local/state}/harnessed/$state_project/$stack/.claude"
```

`$relpath` is already computed at the top of `harnessed_isolated` via `project_relpath`
(home-relative, e.g. `Programming/Personal/code-container`; basename for paths outside $HOME).
Flatten slashes to dashes. The pod/container name (`$instance`/`$pod`) is UNCHANGED — it keeps
the compact hash slug (DNS-label ≤63-char constraint); only the state-dir location becomes legible.

**Pre-existing hash-based state dirs** from earlier runs are left in place (unused); new runs
create the legible path. No migration needed for a dev tool.

**Done when:** `test_legible_slug` passes AND `test_state_persists`/`test_fresh_wipes` still pass
(the persistence logic is path-agnostic; only the helper that computes the dir changed).

### Task 3 — UAT suite tracks the new layout

Update `tools/uat/phase-04.sh`:
- Add `uat_state_dir <stack> <project>` mirroring the launcher's new legible layout.
- `test_state_persists` / `test_fresh_wipes`: compute the state dir via `uat_state_dir` (was the hash formula).
- `test_legible_slug`: assert the dir exists at `uat_state_dir` and its path contains the project token (was a flat glob that caught stale hash dirs).

**Done when:** `./tools/uat/run-uat.sh 4` → 16/16 tests, 50/50 checks, exit 0.
